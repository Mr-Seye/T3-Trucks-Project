# T3 Food Truck Data Platform

This project implements a data platform designed to process and analyse transaction data generated by food trucks. It extracts data from a relational database, transforms it into an optimised format for querying, and presents the results through interactive dashboards and automated reports to support business decision-making.

---

## System Architecture

The platform follows a structured data pipeline:

* Data is sourced from an AWS RDS MySQL database
* An ETL process extracts, transforms, and prepares the data
* The processed data is stored in a time-partitioned S3 bucket
* AWS Athena is used to query the data efficiently
* Insights are delivered via a Streamlit dashboard and automated reporting through AWS Lambda

## Setup Instructions

### Prerequisites

* Python 3.10 or later
* Configured AWS credentials
* Access to the source RDS database

---

### Installation

Create and activate a virtual environment, then install the required dependencies:

```bash
python -m venv .venv
source .venv/bin/activate
pip install -r pipeline/requirements.txt
```

---

### Environment Configuration

Create a `.env` file containing the database connection details:

```env
DB_HOST=rds-host
DB_USER=username
DB_PASSWORD=password
DB_NAME=database-name
DB_PORT=3306

AWS_ACCESS_KEY=your-access-key
AWS_SECRET_KEY=your-secret-key
S3_BUCKET=your-s3-bucket-name
AWS_REGION=your-aws-region

ATHENA_DATABASE=athena-db-name
ATHENA_S3_OUTPUT=s3://your-s3-bucket-name/output/
ATHENA_TX_TABLE=transaction
ATHENA_TRUCK_TABLE=truck
ATHENA_PM_TABLE=payment_method
ATHENA_WORKGROUP=primary


```

## Usage

### Running the ETL Pipeline

To execute incremental data updates:

```bash
cd pipeline
python main.py
```

---

### Launching the Dashboard

To start the Streamlit application:

```bash
cd dashboard
streamlit run app.py
```

---

## Key Features

* **Revenue Monitoring**: Provides insights into total revenue, transaction volume, and performance across individual food trucks
* **Culinary Metrics**: Tracks food safety ratings and compliance indicators
* **Automated Reporting**: Generates daily HTML reports using AWS Lambda
* **Partitioned Data Storage**: Implements Hive-style partitioning to improve query efficiency

---

## Infrastructure

Cloud resources are provisioned and managed using Terraform, including services such as S3, AWS Glue, and Athena.

To deploy the infrastructure:

```bash
cd pipeline/terraform
terraform init
terraform apply
```
